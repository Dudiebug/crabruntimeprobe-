# CrabSyncV2 Research Roadmap

These are RuntimeProbe research phases in the recommended order. Each phase is read-only unless explicitly stated as a future CrabSyncV2-only sandbox. Stop after any crash, stale artifact warning, unexpected gate, or evidence that contradicts the phase contract.

| Phase | Goal | Required config gates | Allowed | Forbidden | Expected evidence file | Success criteria | Crash stop condition | CrabSyncV2 decision unlocked |
|---|---|---|---|---|---|---|---|---|
| 1. Local inventory userdata introspection, metadata only | Inspect wrapper metadata for local inventory arrays. | `allowInventoryUserdataIntrospectionProbes = true`; all write/RPC/deep/Info/HUD/raw identity gates false. | Type, redacted `tostring`, guarded metatable metadata, guarded length metadata. | Traversal, element dereference, `InventoryInfo`, `Enhancements`, writes, RPCs. | `access_evidence_<session>.jsonl`. | Metadata rows for local arrays with no traversal markers. | Any crash after prepare or missing no-traversal/no-dereference markers. | Whether userdata metadata is observable. |
| 2. Local inventory count strategy proof | Prove a count strategy without item reads. | New explicit count gate or reviewed shallow gate; deep/Info/write/RPC false. | Count-only operation against proven parent wrapper. | Element access, item metadata, writes. | `access_evidence_<session>.jsonl`. | Count is stable and scoped, or safely returns nil/error. | Crash or native error around count operation. | Whether array diff can know length. |
| 3. Local first-element wrapper proof | Prove first wrapper can be observed. | Explicit wrapper gate; deep dereference false unless separately reviewed. | Observe wrapper existence only. | Calling `get()`, DA reads, `InventoryInfo`. | `access_evidence_<session>.jsonl`. | First wrapper visibility without dereference. | Any crash or ambiguous parent state. | Whether item iteration can advance. |
| 4. Local first-element dereference proof | Prove one element can be dereferenced. | `allowDeepArrayProbes = true` in a single manual phase. | One capped first-element dereference from proven parent/wrapper. | Loops, multiple elements, `InventoryInfo`, writes. | `access_evidence_<session>.jsonl`. | Valid element or safe nil/error with no crash. | Any crash or invalid/stale object signal. | Whether item objects can be read. |
| 5. Local DA identity read proof | Read item DA identity from dereferenced item. | Deep gate true; Info/write/RPC false. | DA property plus safe `GetName`/`GetFullName` after validation. | `InventoryInfo`, `Enhancements`, writes. | `access_evidence_<session>.jsonl`. | DA identity recorded for first item. | Crash, stale DA object, or unsafe direct field path. | Item identity design. |
| 6. Local InventoryInfo scalar read proof | Read `Level` and `AccumulatedBuff` scalars. | `allowInventoryInfoProbes = true` after element/DA proof. | Capped scalar read from one proven item. | Enhancements traversal, writes. | `access_evidence_<session>.jsonl`. | Scalars read or safe nil/error, with finite/clampable values if present. | Crash or parent mismatch. | Metadata preservation feasibility. |
| 7. Local Enhancements shape/count proof | Inspect Enhancements container shape/count. | `allowInventoryInfoProbes = true`; explicit enhancements metadata phase. | Shape/count only. | Enhancement element values, writes. | `access_evidence_<session>.jsonl`. | Shape/count scoped to proven `InventoryInfo`. | Crash or missing parent proof. | Upgrade metadata strategy. |
| 8. Local Enhancements value proof | Read enhancement values. | Explicit enhancements value gate after shape/count proof. | Capped first value read. | Bulk traversal, writes. | `access_evidence_<session>.jsonl`. | Value representation understood safely. | Any crash or non-finite/ambiguous value. | Full item metadata model. |
| 9. Joined-client read-only replay | Replay each proven local phase as joined client. | Only the gate for the replayed phase; no apply/write. | Same read-only operations already proven locally. | New deeper reads, writes, RPCs. | `access_evidence_<session>.jsonl`. | Joined-client evidence matches local safety without crash. | Any crash, stale session, role instability. | Joined-client reader enablement. |
| 10. Remote PlayerState inventory visibility research | Determine whether remote PlayerStates expose arrays/item metadata. | Identity/resource visibility gates plus the minimal inventory metadata gate under review. | Capped remote visibility checks. | Writes, RPCs, deep traversal beyond proven layers. | `access_evidence_<session>.jsonl`. | Remote inventory metadata is visible or explicitly unresolved. | Crash or raw identity leak. | P2P inventory vs relay/server. |
| 11. Health multiplayer watch | Observe health fields across multiplayer lifecycle. | `allowHealthProbes = true`; no `CrabHC` unless separate discovery. | PlayerState-scoped health samples. | Unscoped `CrabHC`, writes/RPCs. | `access_evidence_<session>.jsonl`. | Stable multiplayer samples by role/context. | Crash or `CrabHC` evidence in playerstate-only phase. | Health merge/pooling inputs. |
| 12. P2P carrier feasibility research | Determine whether CrabSyncV2 can safely piggyback extra state onto a proven player/health/resource sharing path. | Only gates for already proven player/resource/health paths; inventory gates only for separately proven layers. | Read-only inspection of proven paths and metadata-only carrier feasibility notes. | Writes, mutating RPCs, mutating gameplay state, using keys as sync payload, sending inventory payloads through an unproven carrier, claiming inventory sync is proven, inventory element traversal unless separately proven. | `access_evidence_<session>.jsonl` plus carrier feasibility notes. | A safe carrier candidate and payload limits are characterized, or relay/server fallback remains required. | Any crash, lifecycle instability, unexpected mutation, or unproven carrier payload attempt. | P2P piggyback architecture candidate vs external relay fallback. |
| 13. Player-owned CrabHC discovery | Safely identify player-owned health components. | Explicit health discovery gate. | Availability, capped candidates, ownership linkage after review. | Using unscoped first result as player health. | `access_evidence_<session>.jsonl`. | Player ownership relationship proven or unresolved. | Crash or non-player ambiguity without markers. | Optional health component fallback. |
| 14. CrabSyncV2-only RPC/write sandbox design | Design future mutation tests outside RuntimeProbe defaults. | Future CrabSyncV2 safety gates, not RuntimeProbe default. | Manual sandbox planning and explicit apply experiments. | RuntimeProbe mutating RPCs/writes. | Separate CrabSyncV2 sandbox evidence. | Official or raw apply path characterized. | Any crash, save corruption, or UI/OnRep stale state. | Apply executor design. |
